# Symphony-safe Issue Workflow

## 목적

activities 프로젝트에서 Symphony가 논의용 이슈를 실행 작업으로 오해하지 않도록 이슈 보드 운영 규칙을 명확히 한다. Symphony는 `Todo` 또는 `In Progress`에 있는 이슈를 실행 가능한 작업으로 취급하므로, 논의와 실행의 경계를 보드 상태와 이슈 본문에 함께 표시한다.

## 상태별 사용 기준

| 상태 | 용도 | Symphony 처리 기준 |
| --- | --- | --- |
| `Backlog` | 아이디어, 후보 작업, 아직 실행 여부가 확정되지 않은 요구사항 | 실행하지 않는다. 논의와 정리만 가능하다. |
| `Ready` | 논의가 진행 중이거나 요구사항을 다듬는 작업 | 실행하지 않는다. 구현 준비가 끝났더라도 아직 `Todo`가 아니면 코딩하지 않는다. |
| `Todo` | 지금 바로 실행할 수 있고 Symphony가 처리해도 되는 작업 | 실행 대상으로 본다. 한 번에 하나의 이슈만 둔다. |
| `In Progress` | Symphony 또는 사람이 현재 작업 중인 이슈 | 중복 작업을 피하고 현재 작업만 이어간다. |
| `In Review` / `Done` | PR 검토 중이거나 완료된 이슈 | 기본적으로 새 작업을 시작하지 않는다. 명시적인 후속 지시가 있을 때만 처리한다. |

## 논의용 이슈 규칙

논의용 이슈는 `Ready` 또는 `Backlog`에 둔다. `Todo`로 올리지 않는다.

논의용 이슈 제목 또는 본문에는 다음 문구를 명시한다.

```text
Planning Only / Do not implement yet
```

이 문구가 있는 이슈는 요구사항 정리, 설계 검토, 질문 답변, 범위 조정만 허용한다. 코드 작성, 파일 수정, 브랜치 생성, 커밋, PR 생성은 하지 않는다.

## 실행용 이슈 규칙

실행용 이슈만 `Todo`로 승격한다. `Todo`로 올리기 전에 다음 조건을 확인한다.

1. 작업 범위가 명확하다.
2. 수정 가능한 경로와 서비스 경계가 명확하다.
3. 완료 조건과 검증 방법이 적혀 있다.
4. 구현을 시작해도 된다는 의도가 분명하다.
5. 논의용 문구 `Planning Only / Do not implement yet`가 없다.

`Todo`에 있는 이슈는 Symphony가 브랜치 생성, 파일 수정, 검증, 커밋, PR 생성을 수행해도 되는 실행 요청으로 간주한다.

## 한 번에 하나만 Todo로 올리는 원칙

`Todo`에는 한 번에 하나의 실행용 이슈만 둔다.

이 원칙은 Symphony가 여러 작업을 동시에 활성 작업으로 해석해 의도하지 않은 코딩을 시작하는 일을 막기 위한 것이다. 다음 실행 이슈는 현재 `Todo` 또는 `In Progress` 이슈가 PR 생성까지 끝난 뒤에 `Todo`로 승격한다.

```text
Backlog/Ready          Todo              In Progress
논의/대기 이슈 여러 개 -> 실행 이슈 1개 -> 현재 작업 1개
```

## 자동 코딩 방지 규칙

Symphony가 코딩을 시작해도 되는 조건은 다음과 같다.

- 이슈가 실행용 이슈다.
- 이슈가 `Todo` 또는 `In Progress`에 있다.
- 서비스 prefix가 유효하다. 예: `[sns]`, `[recipe]`, `[todo]`
- 작업 범위가 서비스 경계 또는 명시된 shared/root 변경 범위 안에 있다.
- 이슈 본문에 `Planning Only / Do not implement yet`가 없다.

다음 경우에는 코딩하지 않는다.

- 이슈가 `Backlog` 또는 `Ready`에 있다.
- 이슈가 논의용이고 `Planning Only / Do not implement yet`가 적혀 있다.
- `Todo`에 실행 이슈가 둘 이상 있어 우선순위가 모호하다.
- 서비스 prefix가 없거나 지원하지 않는 prefix다.
- 요구사항이 구현 지시가 아니라 설계 논의, 질문, 의사결정 요청이다.

## 운영 체크리스트

논의용 이슈를 만들 때:

- `Backlog` 또는 `Ready`에 둔다.
- 본문 상단에 `Planning Only / Do not implement yet`를 적는다.
- 결정해야 할 질문과 선택지를 적는다.

실행용 이슈로 전환할 때:

- 논의용 문구를 제거한다.
- 완료 조건과 검증 방법을 확정한다.
- 필요한 경우 서비스 prefix를 확인한다.
- 현재 `Todo`가 비어 있는지 확인한다.
- 이슈 하나만 `Todo`로 올린다.

이 규칙을 따르면 논의용 이슈는 논의 상태에 머물고, Symphony는 명시적으로 승격된 실행용 이슈만 처리한다.

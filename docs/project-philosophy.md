업무자동화 프로젝트 철학 및 방향성

1. 프로젝트 목적

이 프로젝트는 단순 AI 챗봇 프로젝트가 아니다.

이 프로젝트의 목표는:

«“활동가와 작은 조직의 반복 운영 업무를 구조화하고 자동화하여,
사람의 기억과 수작업에 의존하지 않는 지속 가능한 운영 시스템을 만드는 것”»

이다.

핵심은 “강력한 AI”가 아니라:

- 업무 흐름 구조화
- 반복 업무 제거
- 조직 맥락 보존
- 인수인계 비용 감소
- AI 사용 전 정리 노동 감소

이다.

---

2. 문제 정의

현재 활동가/시민단체/작은 조직은 다음 문제를 겪는다.

2-1. 업무 정보가 흩어져 있다

정보가 다음 위치에 분산된다.

- 카카오톡
- 텔레그램
- 구글 드라이브
- PDF
- 사진
- 메모
- 음성
- SNS
- 이메일

하지만 이 정보는 “업무 데이터” 상태가 아니다.

즉:

- 검색이 어렵고
- 맥락이 연결되지 않으며
- task로 구조화되지 않는다.

---

2-2. 사람의 기억에 의존한다

조직 운영이:

- 특정 담당자
- 오래 일한 활동가
- 카톡 검색
- 개인 메모

에 의존한다.

그 결과:

- 인수인계가 어렵고
- 담당자가 떠나면 맥락이 사라지며
- 같은 일을 반복하게 된다.

---

2-3. AI 사용 전 정리 작업이 더 오래 걸린다

현재 AI 사용의 실제 병목은:

- 프롬프트 작성
- 문서 정리
- context 설명
- 파일 정리
- 관련 링크 수집

이다.

즉:

«“AI를 사용할 수 있는 상태”를 만드는 비용이 너무 높다.»

---

3. 프로젝트의 핵심 철학

3-1. 우리는 AI보다 workflow를 먼저 해결한다

이 프로젝트의 핵심은:

- 강력한 agent
- 완전 자동화
- AGI

가 아니다.

핵심은:

«“비정형 업무 흐름을 구조화하는 것”»

이다.

AI는 그 이후 단계이다.

---

3-2. 사람의 기존 workflow를 파괴하지 않는다

사용자는:

- 카카오톡을 계속 사용하고
- 텔레그램을 계속 사용하고
- 구글독스를 계속 사용한다.

새로운 업무 방식을 강제하면 실패한다.

따라서 이 프로젝트는:

«“기존 workflow 위에 자동화를 덧붙인다”»

를 원칙으로 한다.

---

3-3. 완전 자동화보다 “보조 자동화”를 우선한다

활동가 업무는:

- 법률
- 사람 관계
- 정치적 맥락
- 감정
- 긴급 대응

등이 포함되므로 완전 자동화가 위험하다.

따라서:

- AI가 초안을 만들고
- 사람이 검토하고
- 사람이 최종 결정한다

는 구조를 기본 원칙으로 한다.

Human in the Loop를 유지한다.

---

3-4. AI보다 조직 기억이 더 중요하다

이 프로젝트의 장기 목표는:

«“사람이 기억하던 조직 맥락을 시스템이 기억하게 만드는 것”»

이다.

Task에는:

- 관련 문서
- 과거 회의
- 담당자
- 링크
- 법률 정보
- 결정 이유
- 이전 사례

가 연결되어야 한다.

---

4. 프로젝트 단계별 목표

Phase 1 — 업무 구조화

목표:

«“비정형 데이터를 업무 데이터로 변환”»

한다.

주요 기능

- 대화 → TODO 추출
- 회의 요약
- 일정 추출
- 영수증 OCR
- 계좌내역 정리
- SNS 멀티 업로드
- 파일 자동 태깅
- PDF 요약
- 음성 → 텍스트

성공 기준

- 반복 보고 시간이 감소한다
- TODO 누락이 감소한다
- 문서 찾는 시간이 감소한다
- AI 사용 전 정리 시간이 감소한다

---

Phase 2 — 조직 운영 메모리

목표:

«“조직 맥락을 시스템이 기억”»

한다.

주요 기능

- Task context 자동 연결
- 관련 문서 추천
- 유사 사례 연결
- 담당자 히스토리
- timeline 기반 업무 흐름
- 조직 knowledge graph

성공 기준

- 인수인계 시간이 감소한다
- 신규 참여자가 빠르게 적응한다
- 과거 결정 이유를 추적 가능하다

---

Phase 3 — AI 업무 보조 Agent

목표:

«“구조화된 조직 위에서 AI가 업무를 보조”»

한다.

주요 기능

- 보고서 초안 생성
- 성명문 draft
- 관련 기사 검색
- 법률 자료 연결
- 회의 agenda 생성
- SNS 초안 생성
- task 추천

원칙

- 사람 승인 없이 중요한 행동 금지
- 민감한 정보는 검토 필수
- AI는 보조자이며 최종 결정자는 사람

---

5. 이 프로젝트가 중요하다고 판단하는 흐름

현재 기술 생태계는:

- 단순 AI 모델 경쟁
  에서
- workflow orchestration
- tool ecosystem
- observability
- async execution
- agent runtime

중심으로 이동하고 있다.

즉:

«“AI가 코드를 잘 쓰는가”
보다
“AI workflow를 어떻게 운영하는가”
가 중요해지고 있다.»

이 프로젝트는:

«“활동가 조직을 위한 AI 운영체제”»

를 목표로 한다.

---

6. 기술 원칙

6-1. 무료에 가까운 구조 우선

우리는:

- 비싼 enterprise SaaS 의존을 최소화한다.
- self-hosted 가능한 구조를 선호한다.
- 작은 조직도 사용할 수 있어야 한다.

---

6-2. 작은 모델 우선

모든 문제를 대형 모델로 해결하지 않는다.

우선:

- OCR
- tagging
- TODO extraction
- classification

같은 작은 문제를 해결한다.

---

6-3. 점진적 자동화

한 번에 모든 업무를 자동화하려 하지 않는다.

우선:

- 반복 업무 제거
- 정리 자동화
- context 구조화

부터 해결한다.

---

7. 개발 우선순위

우선순위는 다음과 같다.

1. 대화 → TODO 변환
2. 영수증/정산 자동화
3. 문서 tagging/search
4. 회의 요약
5. SNS 멀티 업로드
6. context 연결
7. knowledge graph
8. AI task 추천
9. AI 문서 생성
10. agent workflow

---

8. 프로젝트의 핵심 문장

«우리는 “AI가 모든 일을 대신하는 시스템”을 만드는 것이 아니다.»

«우리는 “사람이 반복적인 운영 마찰 없이 더 중요한 활동에 집중할 수 있는 구조”를 만든다.»

«핵심은 AI가 아니라 workflow다.»
